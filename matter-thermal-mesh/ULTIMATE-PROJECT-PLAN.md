## Vision

This is a **training creation platform** that produces professional, reproducible IoT guides.

The first guide (Smart Home Matter Starter with Grid-EYE + Matter) is the **template**. After it works, we use the same system to produce more guides for different devkits, sensors, protocols, and frameworks.

### What Each Guide Has
- Screenshots of every step (auto-generated by Playwright + manual photos)
- Simple instructions a complete beginner can follow
- Both click-through (with screenshots) AND terminal commands
- Testable — Playwright can re-run guide steps and verify they still work
- Generic credentials — no personal data, ready for public training
- Version-tracked — which SW versions were tested and verified

### Future Guides (same template system)
- xG24 Explorer Kit + PIR motion sensor
- xG24 Dev Kit + accelerometer + Zephyr RTOS
- SparkFun Thing Plus + relay actuator
- Nano Matter + air quality sensor (SGP40)
- Simplicity Studio SDK workflows
- Zephyr RTOS on Silicon Labs (top SiLabs priority)
- Any Silicon Labs devkit + any Qwiic/MikroBUS sensor

#### Why SiLabs MCUs? (Not ESP32)

The sensor nodes and Thread RCP use **Silicon Labs EFR32MG24** chips because:
- Best-in-class Thread/Matter certified radios
- Ultra-low power (~3x better than ESP32 in sleep)
- Dedicated 802.15.4 radio (not shared with WiFi)
- Secure Vault (hardware root of trust)
- The sensor/actuator (Grid-EYE, SCD40, any I2C device) can be any brand — they're interchangeable. The MCU and radio is what matters.

#### Sensor/Actuator Alternatives (Swap Freely)

| Instead of | You can use | Price | Notes |
|-----------|------------|------:|-------|
| Grid-EYE AMG8833 ($22) | MLX90640 ($25-35) | Higher res (32x24) but more expensive |
| Grid-EYE AMG8833 ($22) | PIR motion sensor ($2-5) | Binary presence only, no heatmap |
| SCD40 ($45) | BME280 ($3-5) | Temp+humidity only, no CO2 |
| SCD40 ($45) | SGP40 ($8-12) | VOC air quality, no CO2 |
| Qwiic OLED ($25) | Generic SSD1306 ($4-6) | Needs soldering, no Qwiic |
| IS31FL3741 ($15) | WS2812B strip ($5-8) | Different wiring, no I2C |

All sensors connect via I2C (Qwiic) — swap any sensor without changing the MCU board or radio.

## Guide Design Principles

1. **Beginner-first:** Every step is explicit. No assumed knowledge.
2. **Click + Terminal:** Primary path uses screenshots (click here, type here). Terminal commands shown as "Advanced" alternative.
3. **Generic credentials:** user `smartpi`, password `smartpi`, hostname `smartpi.local`
4. **Screenshot at every step:** Playwright captures HA/dashboard automatically, manual screenshots for Arduino IDE and hardware
5. **Testable:** Playwright test suite verifies guide steps work end-to-end
6. **Version-tracked:** Every guide states which SW versions were tested
7. **Platform notes:** Tested on macOS, Windows differences noted where applicable
8. **Modular:** Swap devkits/sensors/protocols without rewriting the whole guide
9. **English:** All documentation, guides, and content in English

## Sustainability & Maintenance

### Version Matrix

Every guide includes a tested version matrix:
```
| Component          | Tested Version | Min Version |
|-------------------|---------------|-------------|
| Arduino IDE        | 2.3.4         | 2.0.0       |
| SiLabs Arduino Core| 3.0.0        | 3.0.0       |
| Home Assistant     | 2026.3        | 2024.1      |
| OTBR Docker        | latest        | -           |
| Python             | 3.12          | 3.9         |
```

### Update Process

When new versions come out (new Arduino IDE, new SiLabs core, etc.):
1. Update version matrix
2. Re-run Playwright tests to verify guide still works
3. Update screenshots if UI changed
4. Update CHANGELOG.md
5. Tag new release

### GitHub Issue Templates

Structured bug reports for efficient debugging:
- "Guide step not working" — which step, what error, which versions
- "Firmware build error" — which board, Arduino IDE version, error log
- "Hardware connection issue" — which sensor, which board, I2C scan output
- "Feature request" — for new guides or improvements

## Risk Mitigations

| Risk | Mitigation |
|------|-----------|
| DTR issue with XIAO MG24 | RPi OS Docker OTBR likely avoids this; socat fallback ready |
| Matter commissioning fails | Reset boards from Mac via Commander/OpenOCD; phone app fallback |
| OTBR can't find XIAO | `/dev/ttyACM0` device mapping in docker-compose |
| 7" display not detected | RPi OS Desktop auto-detects DSI; `/boot/config.txt` fallback |
| WiFi IP changes | Use `smartpi.local` (mDNS) everywhere |
| Playwright can't reach HA | Direct network access from Mac to RPi |
| New SW version breaks guide | Playwright test suite catches regressions |
